# Ubuntu上部署Ghost

请根据笔记的写作日期，选择性阅读。

作者： 放羊的大少爷
Home： joynoting.com
写作日期： 2018-11-19
系统： Ubuntu@18.04
软件版本：
Nodejs@v10.14.1
Ghost-CLI@1.9.8
Ghost@2.6.2
Nginx@1.14.0
参考：
Ghost Documentation

阅读本篇笔记前，建议先阅读另一篇笔记《使用Ghost本地搭建个人博客》。

注意：因为这篇笔记距离 《本地搭建个人博客~Ghost》 有一段时间，所以各软件版本存在差异，但是不影响本篇笔记所述的安装方法。

本次搭建使用的机器是在腾讯云购买的CVM，本篇着重讲解在 Ubuntu 上为 Ghost 搭建做的前提准备以及 Nginx 配置。

# 1. 查看系统版本

# 2. 安装curl

>> sudo apt install curl

# 3. 安装Nodejs

与本地搭建 Nodejs 一样，需要查看 Ghost 所支持的 Nodejs 版本，选择兼容的 Nodejs 版本进行安装。

我这里选择的是 v10.14.1 版本。

默认情况下 apt-get 安装的版本不确定，所以需要提前设置指定版本。

>> curl -sL https://deb.nodesource.com/setup_10.x | sudo -E bash -
1

然后安装 Nodejs

>> sudo apt-get install -y nodejs
1

版本列表可以在这里查看： https://github.com/nodesource/distributions/tree/master/deb 然后自行修改参数 setup_10.x

安装完成后查看自己的 Nodejs 版本

>> node -v
v10.14.1
1
2

# 4. 安装Nginx

>> sudo apt-get install nginx
1

# 二、安装Ghost

现在开始正式安装 Ghost，我采用的是全局安装 Ghost-CLI 的方式进行本地安装，这只是安装方式的一种，除此之外，你还可以选择 git 源码进行安装。

点击查看 官方源码

安装步骤如下：

  1. 全局安装 Ghost-CLI
>> sudo npm install ghost-cli -g
1

注意：如果不以管理员权限运行，将会出现错误信息。

如果已经安装，想要升级 ghost-cli，可执行以下命令。

>> sudo npm install -g ghost-cli@latest
1
  1. 本地安装Ghost

Ghost-CLI 安装完成后，cd 到一个空目录下安装 Ghost，该目录就是 Ghost 的安装目录。

注意： 以后执行 ghost start 命令需要 cd 到该目录下执行。

>> ghost install local
1

# 三、Nginx配置

  1. 删除默认的 Nginx 全部配置文件
>> sudo rm /etc/nginx/sites-available/default
>> sudo rm /etc/nginx/sites-enabled/default
>> sudo rm /etc/nginx/conf.d/default
1
2
3
  1. 新建配置文件
>> sudo touch /etc/nginx/sites-available/ghost.conf
1

添加文件的权限

>> sudo chmod 777 ghost.conf
1
  1. 以我的域名 pymilk.com为例，配置文件 ghost.conf 的内容如下：
server {
    listen 80;
    server_name pymilk.com;
    location / {
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_set_header   Host      $http_host;
        proxy_pass         http://127.0.0.1:2368;
    }
}
1
2
3
4
5
6
7
8
9
  1. 创建文件的软链接

此时文件 ghost.conf 还未生效，需要把它软链接到 sites-enabled 中:

>> sudo ln -s /etc/nginx/sites-available/ghost.conf /etc/nginx/sites-enabled/ghost.conf
1
  1. 重启Nginx服务器

至此 Nginx 已配置完成，重启Nginx服务器即可。

>> sudo service nginx restart
1
  1. 访问主站

至此已经全部搭建完成，不出意外即可直接访问主机 IP。
